Qualia-Urban

This project is an extension of the Qualia project which aims to allow a Qualia Engine to handle many more different types of data. It is initially being developed for the upcoming MediaCities conference along with collaborators in DAT 203.

API Draft Specification

Resources

Resources allow access to data stored within the Engine. Performing a GET request of the URL root will return a list of those types of objects. They are filterable by passing options into the query string.

Things

Things are physical objects. They have a location, which can be updated over time, and a list of channels, which take readings.

http://urban.qualia.org.uk/api/v3/thing/?api_key=API_KEY

Collectors

Collectors allow new data to be submitted to the engine. At present, the user can POST JSON data to the following URLs. The examples are of the expected structure of the JSON data. It is envisaged that this data can be sent to the engine over HTTPS, MQTT, OSC and WebSockets.

Location

http://urban.qualia.org.uk/api/v3/collector/location/

{
  "api_key": "API_KEY",
  "thing": 1,
  "location":{
    "coordinates": [
      -4.130859374424982,
      19.88814027046907
    ],
    "recorded": "2014-10-23T00:01:05.569542",

  }
}
